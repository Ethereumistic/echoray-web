## üåä Brand Philosophy

### Origin Story
Echoray.io draws inspiration from **Mr. Ray** in Finding Nemo‚Äîthe wise educator who brings clarity to the vast, complex underwater world. Just as Mr. Ray guides young fish through the ocean's mysteries, **Echoray.io illuminates the digital landscape for businesses navigating the web**.

**The Ocean Metaphor:**
- **The Ocean** = The vast, overwhelming internet ecosystem (Web2, Web3, digital infrastructure)
- **Mr. Ray** = Echoray.io‚Äîthe beacon of knowledge, clarity, and guidance
- **Dory** = Security-naive users who reuse passwords, need protection
- **Nemo** = Curious explorers who may stumble into digital threats
- **Bruce** = Gray-hat hackers, ethical dilemmas in the digital space

### Mission
**We bring clarity and understanding to the web's complexity.** Echoray.io serves businesses ranging from simple brochure sites to enterprise-grade custom CRMs, helping them thrive in the ever-evolving digital landscape.

### Color Psychology & Application

**When to use each shadcn/ui variant:**

**`<Button variant="default">`** ‚Üí Primary actions, main CTAs
**`<Button variant="secondary">`** ‚Üí Secondary actions, less prominent
**`<Button variant="outline">`** ‚Üí Tertiary actions, alternative choices
**`<Button variant="destructive">`** ‚Üí Delete, remove, dangerous actions
**`<Button variant="ghost">`** ‚Üí Subtle actions, icon buttons
**`<Button variant="link">`** ‚Üí Text-style navigation

**`<Badge variant="default">`** ‚Üí Primary status (active, live)
**`<Badge variant="secondary">`** ‚Üí Neutral status (pending)
**`<Badge variant="outline">`** ‚Üí Subtle emphasis
**`<Badge variant="destructive">`** ‚Üí Error, failed, critical

**`<Alert variant="default">`** ‚Üí General information
**`<Alert variant="destructive">`** ‚Üí Errors, critical warnings

## üìê Spatial System

### Border Radius
```css
--radius: 0.5rem; /* 8px - Professional B2B sweet spot */
```

**Shadcn/ui components automatically use this via:**
- `rounded-lg` ‚Üí Most components (Button, Card, Input)
- `rounded-md` ‚Üí Smaller elements (Badge, smaller buttons)
- `rounded-full` ‚Üí Avatar, circular buttons

### Spacing Scale
```css
--spacing: 0.3rem; /* 4.8px base unit */
```

**Application in shadcn/ui:**
- Use Tailwind spacing classes: `p-4`, `gap-6`, `space-y-8`
- Component internal padding: `p-4` to `p-6`
- Section spacing: `py-12` to `py-24`
- Container padding: `px-4 md:px-6 lg:px-8`

**Recommended spacing scale:**
- Tight: `gap-2` (8px)
- Normal: `gap-4` (16px)
- Comfortable: `gap-6` (24px)
- Generous: `gap-8` (32px)
- Section: `gap-12` (48px)

### Shadow System
```css
--shadow-color: #1a1a1a;
--shadow-opacity: 0.16;
```

**Shadcn/ui components with built-in shadows:**
- `<Card>` ‚Üí Subtle elevation
- `<Dialog>` ‚Üí Modal elevation
- `<Popover>` ‚Üí Floating UI elevation
- `<DropdownMenu>` ‚Üí Menu elevation
- `<HoverCard>` ‚Üí Tooltip elevation

**DO NOT** add custom shadows. Use shadcn/ui's default elevation system.

#### 7. Contact Form with React Hook Form

```tsx
"use client"

import { zodResolver } from "@hookform/resolvers/zod"
import { useForm } from "react-hook-form"
import * as z from "zod"
import { Button } from "@/components/ui/button"
import { Form, FormControl, FormDescription, FormField, FormItem, FormLabel, FormMessage } from "@/components/ui/form"
import { Input } from "@/components/ui/input"
import { Textarea } from "@/components/ui/textarea"
import { Select, SelectContent, SelectItem, SelectTrigger, SelectValue } from "@/components/ui/select"
import { toast } from "@/components/ui/use-toast"

const formSchema = z.object({
  name: z.string().min(2, "Name must be at least 2 characters"),
  email: z.string().email("Invalid email address"),
  projectType: z.string(),
  message: z.string().min(10, "Message must be at least 10 characters"),
})

export function ContactForm() {
  const form = useForm<z.infer<typeof formSchema>>({
    resolver: zodResolver(formSchema),
    defaultValues: {
      name: "",
      email: "",
      projectType: "",
      message: "",
    },
  })

  function onSubmit(values: z.infer<typeof formSchema>) {
    console.log(values)
    toast({
      title: "Form submitted!",
      description: "We'll get back to you within 24 hours.",
    })
  }

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-6">
        <FormField
          control={form.control}
          name="name"
          render={({ field }) => (
            <FormItem>
              <FormLabel>Name</FormLabel>
              <FormControl>
                <Input placeholder="John Doe" {...field} />
              </FormControl>
              <FormMessage />
            </FormItem>
          )}
        />

        <FormField
          control={form.control}
          name="email"
          render={({ field }) => (
            <FormItem>
              <FormLabel>Email</FormLabel>
              <FormControl>
                <Input type="email" placeholder="john@company.com" {...field} />
              </FormControl>
              <FormDescription>
                We'll never share your email.
              </FormDescription>
              <FormMessage />
            </FormItem>
          )}
        />

        <FormField
          control={form.control}
          name="projectType"
```

## üéì Final Instructions for AI Models

When building ANY component for echoray.io:

1. **Read this design system first** - Understand the brand and available components
2. **Check shadcn/ui docs** - https://ui.shadcn.com/docs/components
3. **Import shadcn/ui components** - Never write custom components
4. **Compose with Tailwind** - Use utility classes for layout only
5. **Use component variants** - Don't override with custom styles
6. **Ensure responsiveness** - Use Tailwind responsive classes
7. **Trust accessibility** - shadcn/ui handles it via Radix UI
8. **Test in both themes** - Dark (default) and light
9. **Follow brand tone** - Professional yet approachable
10. **Optimize for conversion** - Clear CTAs, trust signals, minimal friction

Remember: Echoray.io brings **clarity to complexity**. Your components should embody this by being clean, semantic, and built from proven shadcn/ui primitives.
